How this works

When you hit Refresh, this app looks up the HTTP IP seen by the server with ip-api.com. If your browser also has IPv6, that address is looked up separately so the page can compare IPv4 and IPv6 routing. Browser fingerprint data is computed entirely in your browser and is not sent back to this server. WebRTC uses a public STUN server to inspect the ICE candidates your browser exposes.

What we don't do

This app has no database and the application code does not write request logs. The hosting platform may still keep operational logs. It does not set cookies or run ad trackers. Page visit counts are recorded via Umami's cookieless analytics. Every check runs on demand.

Data sources

IP geolocation & proxy detection: ip-api.com
Routing cross-check: Cloudflare trace
IPv6 detection: ipv6.icanhazip.com
WebRTC: Google STUN (stun.l.google.com)
